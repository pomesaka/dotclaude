# ADeT 週次定例ミーティング

## 概要

- **頻度**: 毎週月曜日
- **目的**: 先週の進捗確認・今週のタスク調整・ブロッカー解消
- **対象リポジトリ**: Accel-Hack/ADeT, ADeT-AI, ADeT-infra
- **プロジェクトボード**: https://github.com/orgs/Accel-Hack/projects/2/views/7

## ミーティング前の準備

### 1. 週次レポートを生成する

定例開始前に `/weekly-standup` を実行して統計レポートを生成する。

```
/weekly-standup
```

生成される情報:
- 先週のPR開始・マージ数
- 現在オープン中のPR（レビュー待ち含む）
- 先週のIssue作成・クローズ数
- アクティブ貢献者ランキング
- プロジェクトボードの進行中タスク

### 2. ボードを確認・整理する

定例前にボードのステータスを最新状態にしておく:
- 完了タスク → **Done**
- 着手済み → **In progress**
- PRレビュー中 → **Review**

---

## アジェンダ

### 1. 先週のふりかえり（10分）

週次レポートを見ながら確認:

- **完了したこと** — マージされたPR・クローズされたIssue
- **進捗状況** — In progress のタスクの現状
- **滞留の確認** — 7日以上オープンのPRがあれば理由を確認

### 2. 今週の計画（15分）

- **優先タスクの確認** — Todoの中から今週着手するものを決める
- **担当者のアサイン** — アサイン漏れがあれば割り当て
- **ブロッカーの共有** — 依存関係・外部要因・技術的な詰まりどころ

### 3. チケット作成・調整（5分）

新しく浮上したタスクがあれば `/create-ticket` でその場で起票する。

```
/create-ticket [タイトル]
```

### 4. その他（5分）

- リリース予定・マイルストーン確認
- チームへの共有事項

---

## スキル一覧

| スキル | 用途 | 呼び出し方 |
|--------|------|------------|
| `/weekly-standup` | 週次レポート生成 | 毎週月曜の定例前 |
| `/create-ticket` | Issue作成 + ボード追加 | タスクが生まれたとき随時 |
| `/pr-report` | 半期集計レポート | 半期評価時（`/pr-report 2025-10-01 2026-03-31`）|
| `/changelog` | 変更ログ生成 | リリース前・振り返り時 |

---

## プロジェクトボードのステータス

| ステータス | 意味 |
|-----------|------|
| **Todo** | 未着手。優先順位付け待ち |
| **Draft** | 仕様検討中・着手前の調査段階 |
| **In progress** | 開発中 |
| **Review** | PRレビュー待ち |
| **BE done** | バックエンド完了、フロントエンド作業中 |
| **Wait** | 外部依存・ブロック状態 |
| **Done** | 完了（マージ済み） |
| **Abandon** | キャンセル・優先度低でスコープアウト |
| **History** | 過去の記録として残すもの |
