# ドキュメント更新スキル

このセッションでの作業内容と会話を分析し、CLAUDE.mdや.claude/rules/の更新を提案してください。

## 分析対象

### 1. 指摘されたミス・修正依頼
セッション中にユーザーから指摘された内容を抽出：
- 「〇〇は使わないで」「〇〇は禁止」
- 「〇〇を使って」「〇〇の方がいい」
- 「このプロジェクトでは〇〇」
- コーディングスタイルや規約に関する指摘
- ワークフローに関する指摘（git/jjの使い方など）

これらは **Coding Constraints** や **rules** に追記すべき内容です。

### 2. 新しい知見・変更
セッション中に発生した変更を抽出：
- 新しいライブラリやツールの追加
- アーキテクチャやディレクトリ構造の変更
- 新しい開発コマンドやワークフロー
- バージョン更新

これらは **CLAUDE.md** の該当セクションに追記すべき内容です。

## 出力形式

### Step 1: 分析結果の提示
以下の形式で分析結果を提示：

```
## 検出された指摘・学習内容

### ミス・修正指摘
- [ ] 〇〇は禁止 → Coding Constraintsに追記
- [ ] 〇〇を使うべき → rulesに追記

### 新しい知見
- [ ] ライブラリXをv1.2.3に更新 → Main Librariesを更新
- [ ] 新コマンド追加 → Common Development Commandsに追記
```

### Step 2: ユーザー確認
更新内容をユーザーに確認し、承認を得る。

### Step 3: 更新実行
承認された内容のみを適切なファイルに追記：
- プロジェクトのCLAUDE.md
- プロジェクトの.claude/rules/*.md
- 必要に応じて新しいrulesファイルを作成

## 注意事項
- 既存の内容と重複しないよう確認する
- 追記する内容は簡潔に、WHYを含める
- rulesファイルは適切なpathsを設定する
