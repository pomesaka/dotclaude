# スキル フロントマター リファレンス

## フィールド一覧

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `name` | No | スラッシュコマンド名。省略時はディレクトリ名。小文字英数字+ハイフン、最大64文字 |
| `description` | 推奨 | スキルの説明。Claudeの自動呼び出し判定に使われる |
| `argument-hint` | No | オートコンプリートで表示されるヒント。例: `[issue-number]` |
| `disable-model-invocation` | No | `true` でClaude自動呼び出しを無効化 |
| `user-invocable` | No | `false` でスラッシュメニューから非表示 |
| `allowed-tools` | No | スキル実行中に許可なく使えるツール。例: `Read, Grep, Bash(npm *)` |
| `context` | No | `fork` でサブエージェント内で実行（独立コンテキスト） |
| `agent` | No | `context: fork` 時のエージェント種別。`Explore`, `Plan`, `general-purpose` |

## 利用可能な変数

| 変数 | 説明 |
|------|------|
| `$ARGUMENTS` | 全引数（スペース区切り） |
| `$0`, `$1`, ... | N番目の引数（0始まり） |
| `${CLAUDE_SKILL_DIR}` | SKILL.mdがあるディレクトリパス |
| `${CLAUDE_SESSION_ID}` | セッションID |

## 動的コンテキスト注入

バッククォートと `!` でコマンド実行結果を注入できる:

```markdown
## 現在のdiff
!`jj diffu`
```

スキル読み込み時にコマンドが実行され、出力がプロンプトに埋め込まれる。

## プロンプトキャッシュの最適化

`context: fork` で起動したエージェントのプロンプトには、Anthropicのエフェメラルキャッシュ（TTL 5分）が適用される。キャッシュはプレフィックスの一致で判定されるため、**変動するコンテンツをプロンプトの末尾に置く**ことでキャッシュヒット率が上がる。

**設計パターン:**

```
[static] 参照ドキュメント（!cat で埋め込み）
[static] 手順・ルール
[variable] !`コマンド出力`（diff・ログ等）← 末尾に配置
```

**メリット:**
- ツール呼び出し（Read×N回）を省けるためレイテンシ削減
- static部分（ドキュメント＋手順）が2回目以降のループで ~10% のコストになる

**適用例（review系スキルをloopから繰り返し呼ぶケース）:**

```markdown
## レビュー基準
!`cat ~/.claude/skills/review-code/criteria.md`

## 手順
...

## 変更差分  ← 末尾に配置
!`jj diffu -r main..@`
```

静的なドキュメントを `Read` ツールで読むのではなく `!cat` で埋め込むことで、ツール呼び出しフェーズより前（キャッシュ対象のプレフィックス）に静的コンテンツを配置できる。
