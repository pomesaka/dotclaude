# Summarize

現在のセッション状態を構造化し、`/clear` 後の新しいセッションで即座に作業を再開できる引き継ぎメモを生成する。

## 引数の扱い

`$ARGUMENTS` の有無で動作が変わる:

| 状態 | 動作 |
|------|------|
| 引数なし | セッション全体をバランスよくサマライズ |
| 引数あり（例: `auth refactor`） | その観点・作業を**最重要セクションとして最初に展開**し、他は簡潔に補足 |
| `focus on X` 形式 | X を重点サマライズ。他のスレッドは最小限 |
| `drop Y` 形式 | Y に関する内容は省略 |

引数が指定されている場合、引き継ぎメモを受け取った次のセッションが「何をすればいいか」が一瞬でわかる構成にする。

## 手順

### Step 1: セッション状態の把握

現在の会話を振り返り、以下を把握する:

- **Goal**: 何を達成しようとしているか
- **Progress**: どこまで進んでいるか
- **Key Findings**: 何を発見・確認したか
- **Next Steps**: 次にやるべきこと（`$ARGUMENTS` があればここを最優先で詳細に書く）
- **Relevant Files**: 作業に関連するファイル
- **Constraints / Gotchas**: 重要な制約や落とし穴

### Step 2: 引き継ぎメモの生成

新しいセッションの最初のプロンプトとして自己完結した内容にする。`$ARGUMENTS` がある場合は該当セクションを冒頭に置き、詳細に展開する:

````markdown
## 引き継ぎメモ ({today})
{$ARGUMENTS があれば: > Focus: {引数の内容}}

**Goal**: {達成しようとしていること}

**Progress**: {現在地・どこまで進んだか}

**Key Findings**:
- {発見1}
- {発見2}

**Next Steps**:
- [ ] {次にやること1}  ← $ARGUMENTS がある場合、これを最も詳細に書く
- [ ] {次にやること2}

**Relevant Files**:
- `{path}` — {なぜ重要か}

**Constraints / Gotchas**:
- {重要な制約・落とし穴}
````

### Step 3: ファイルへの書き出し

生成した引き継ぎメモを `~/.claude/tmp/handoff.md` に Write ツールで書き出す（上書き、常に最新1件）。

### Step 4: 次のアクションを案内

サマリーを表示した上で、以下を伝える:

```
~/.claude/tmp/handoff.md に保存しました。

次のステップ:
  /clear

新しいセッションの最初に:
  @~/.claude/tmp/handoff.md を読んで、作業を再開して
```

---

## Gotchas

- **`context: fork` にしてはいけない**: フォークすると親セッションの会話が見えず、空のサマリーになる。メインコンテキストで実行すること。
- **handoff.md は上書きされる**: 複数を保持したい場合はタイムスタンプ付きファイル名（`handoff-20260421.md`）を提案する。
